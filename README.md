# ระบบจัดการสต็อกสินค้า (Inventory Management System)

ระบบจัดการสต็อกสินค้าสำหรับธุรกิจขนาดเล็กถึงกลาง พัฒนาโดยทีมพัฒนาซอฟต์แวร์เป็นส่วนหนึ่งของโครงการเรียนการสอน

## คุณสมบัติหลัก

ระบบมีฟีเจอร์หลักตามเป้าหมายของสปรินต์ที่ 1 ดังนี้:

1. **ดูสต็อกทั้งหมด** - พนักงานสามารถเรียกดูสต็อกสินค้าทั้งหมดได้ถูกต้องครบทุกรายการภายในคำสั่งเดียว
2. **เพิ่มสินค้าใหม่** - พนักงานสามารถเพิ่มสินค้าใหม่เข้าระบบได้โดยไม่มีรหัสซ้ำหรือจำนวนติดลบหลุดเข้าไป
3. **ปรับยอดรับเข้า/จ่ายออก** - พนักงานสามารถปรับยอดรับเข้า/จ่ายออกได้ถูกต้อง โดยยอดคงเหลือต้องไม่ติดลบเด็ดขาด
4. **ค้นหาสินค้า** - ผู้จัดการสามารถค้นหาสินค้าด้วยชื่อหรือรหัสแล้วเจอผลลัพธ์ที่ถูกต้องภายในคำสั่งเดียว
5. **ส่งออกรายงานสต็อก** - ผู้จัดการสามารถส่งออกรายงานสต็อกเป็น CSV ที่เปิดใช้ทำบัญชีต่อได้ทันที
6. **การทดสอบ** - ทุก user story มี unit test ครอบ acceptance criteria ทั้งกรณีปกติและ edge case
7. **เอกสารประกอบ** - สมาชิกใหม่สามารถ clone repo แล้วรันโปรแกรมได้เองโดยไม่ต้องถามใคร (เรื่องนี้)

## การติดตั้ง

เนื่องจากโครงการนี้ยังอยู่ในขั้นเริ่มต้น การติดตั้งอาจแตกต่างกันไปตามเทคโนโลยีที่เลือกใช้ โปรดดูไฟล์เอกสารเฉพาะเทคโนโลยีในโฟลเดอร์ `docs/` เมื่อมีการพัฒนาต่อไป

### ขั้นตอนทั่วในการติดตั้ง

1. โคลนรีโพซิทอรี:
   ```bash
   git clone https://github.com/chonnawee-ya/team-03-inventory.git
   cd team-03-inventory
   ```

2. ติดตั้งขึ้นอยู่กับเทคโนโลยีที่ใช้:
   
   **สำหรับ Python (ถ้ามี):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # บน Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

   **สำหรับ Node.js (ถ้ามี):**
   ```bash
   npm install
   ```

   **สำหรับ Java (ถ้ามี):**
   ```bash
   ./mvnw install  # หรือ ./gradlew build
   ```

3. ตั้งค่าตัวแปรแวดล้อม (ถ้าจำเป็น):
   ```bash
   cp .env.example .env
   # แก้ไขไฟล์ .env ตามความเหมาะสม
   ```

## การใช้งาน

เมื่อติดตั้งเสร็จสิ้น คุณสามารถใช้งานระบบได้ผ่านคำสั่งต่อไปนี้:

### 1. ดูสต็อกทั้งหมด
```bash
# ตัวอย่างคำสั่ง (อาจแตกต่างกันไปตามการใช้งานจริง)
python main.py list
# หรือ
npm run list
# หรือ
java -jar inventory.jar list
```

### 2. เพิ่มสินค้าใหม่
```bash
# ตัวอย่างคำสั่ง (อาจแตกต่างกันไปตามการใช้งานจริง)
python main.py add --name "สินค้าตัวอย่าง" --sku "SKU001" --quantity 10
# หรือ
npm run add -- --name "สินค้าตัวอย่าง" --sku "SKU001" --quantity 10
# หรือ
java -jar inventory.jar add --name "สินค้าตัวอย่าง" --sku "SKU001" --quantity 10
```

### 3. ปรับยอดรับเข้า/จ่ายออก
```bash
# ตัวอย่างคำสั่ง (อาจแตกต่างกันไปตามการใช้งานจริง)
python main.py update --sku "SKU001" --quantity-change 5
# หรือ
npm run update -- --sku "SKU001" --quantity-change 5
# หรือ
java -jar inventory.jar update --sku "SKU001" --quantity-change 5
```

### 4. ค้นหาสินค้า
```bash
# ตัวอย่างคำสั่ง (อาจแตกต่างกันไปตามการใช้งานจริง)
python main.py search --query "สินค้า"
# หรือ
npm run search -- --query "สินค้า"
# หรือ
java -jar inventory.jar search --query "สินค้า"
```

### 5. ส่งออกรายงานสต็อก
```bash
# ตัวอย่างคำสั่ง (อาจแตกต่างกันไปตามการใช้งานจริง)
python main.py export --format csv --output inventory.csv
# หรือ
npm run export -- --format csv --output inventory.csv
# หรือ
java -jar inventory.jar export --format csv --output inventory.csv
```

## การมีส่วนร่วม

เรายินดีต้อนรับการมีส่วนร่วมจากทุกคน! โปรดทำตามขั้นตอนต่อไปนี้:

1. ฟอร์ก repository นี้
2. สร้าง branch ใหม่สำหรับฟีเจอร์ของคุณ (`git checkout -b feature/your-feature-name`)
3. ทำการเปลี่ยนแปลงของคุณ
4. ทำการ commit การเปลี่ยนแปลงของคุณ (`git commit -am 'เพิ่มฟีเจอร์ใหม่'`)
5. ผลักไปยัง branch ของคุณ (`git push origin feature/your-feature-name`)
6. สร้าง Pull Request ไปยัง branch `main`

กรุณาตรวจสอบให้แน่ใจว่า:
- โค้ดของคุณผ่านการทดสอบทั้งหมด
- คุณได้อ่านและเข้าใจการเปลี่ยนแปลงที่ทำ
- ข้อความ commit ตรงกับการเปลี่ยนแปลงจริง
- คุณใช้เครื่องมือ AI ที่ไม่มีค่าใช้จ่ายเท่านั้น (ตามนโยบายการใช้ AI ของทีม)

## นโยบายการใช้ AI

ทีมอนุญาตให้ใช้ AI ช่วยในการ:
- เขียนโค้ดร่าง (draft code)
- เขียนข้อความ commit ร่าง (draft commit message)

แต่มีข้อห้ามดังต่อไปนี้:
- คัดลอกโค้ดจาก AI โดยไม่ได้อ่านและทำความเข้าใจก่อน
- ต้องอ่านและแก้ไขข้อความ commit ที่ AI สร้างให้ตรงกับการเปลี่ยนแปลงจริงก่อนทำการ commit
- ใช้เฉพาะ AI ที่ไม่มีค่าใช้จ่าย ไม่บังคับซื้อ subscription

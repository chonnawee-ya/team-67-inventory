## Branching Strategy

ทีมใช้ GitHub Flow:
- main branch ต้อง deploy ได้เสมอ ห้าม commit โดยตรง
- ทุก feature ใหม่ต้องสร้าง branch ชื่อ feat/<issue-number>-<short-name>
- ทุก PR ต้องมีคนอื่นในทีมอย่างน้อย 1 คน review และ approve ก่อน merge

## Sprint Goal (Sprint 1)
- Task 1: "จัดเตรียมฐานข้อมูลจำลองในรูปแบบ JSON ที่มีโครงสร้างถูกต้องและพร้อมใช้งาน เพื่อรองรับการพัฒนาฟังก์ชันจัดการสินค้าในขั้นตอนต่อไปได้อย่างราบรื่น"
- Task 2: "ช่วยให้ผู้ใช้สามารถตรวจสอบรายการสินค้าคงเหลือในสต็อกทั้งหมดได้อย่างชัดเจนผ่านหน้าจอ Command-line พร้อมระบบแจ้งเตือนที่ปลอดภัยรองรับกรณีไม่มีสินค้าในระบบ"
- Task 3: "เพิ่มความสะดวกให้ผู้ใช้สามารถค้นหาข้อมูลสินค้าที่ต้องการได้อย่างรวดเร็ว แม่นยำ และยืดหยุ่น โดยรองรับการค้นหาผ่านรหัสหรือบางส่วนของชื่อสินค้า" 
- Task 4: "ส่งมอบระบบจัดการคลังสินค้าบน Command-line ที่สมบูรณ์ ใช้งานง่ายผ่านเมนูหลัก และผ่านการทดสอบคุณภาพโค้ดก่อนนำขึ้น GitHub เพื่อพร้อมสำหรับการตรวจสอบร่วมกันในทีม"
- Task 5: "พัฒนาหน้าจอเพิ่มสินค้าใหม่พร้อมระบบ Validation ที่ตอบสนองทันที เพื่อให้มั่นใจว่าผู้ใช้งานจะกรอกข้อมูลที่จำเป็นครบถ้วนและรับรู้สถานะการบันทึกได้อย่างชัดเจน"
- Task 6: "ช่วยให้ผู้ใช้สามารถเพิ่มสินค้าใหม่เข้าสู่ระบบได้อย่างถูกต้องและปลอดภัย ผ่านหน้าจอที่ใช้งานง่ายพร้อมระบบตรวจสอบข้อมูลที่แม่นยำทั้งฝั่งหน้าบ้านและหลังบ้าน"

## AI Usage Policy

- ใช้ AI ช่วยเขียน draft code และ draft commit message ได้
- ทุก commit message ที่ AI generate ต้องอ่านและแก้ให้ตรงกับ diff จริงก่อน commit
- ห้าม copy code จาก AI โดยไม่อ่านและทำความเข้าใจก่อน
- ใช้เฉพาะ AI ที่ไม่มีค่าใช้จ่าย ไม่บังคับซื้อ subscription
